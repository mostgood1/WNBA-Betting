# Props Files Explanation

## Question
> "We have a props recommendations file but do we have an actual props model for all player stats file?"

## Answer: YES - It's `props_edges_YYYY-MM-DD.csv`

The **`props_edges_YYYY-MM-DD.csv`** file contains all player prop model predictions combined with odds and edge calculations. This is your comprehensive props model output file.

## Props File Types

### 1. `props_predictions_YYYY-MM-DD.csv` (Raw Model Output)
**What it contains:**
- Player name, team
- Model predictions for each stat: PTS, REB, AST, THREES, PRA
- Raw predicted values (e.g., "Klay_predicted_pts": 18.5)
- Calibration adjustments applied

**When it's created:**
- Generated by `predict-props` command
- Uses ONNX models with NPU acceleration (5 models: pts, reb, ast, threes, pra)
- Requires either scoreboard slate OR historical parquet data

**Example columns:**
```
player_name, team, home, opponent, pts, reb, ast, threes, pra, ...
```

### 2. `props_edges_YYYY-MM-DD.csv` ⭐ **THIS IS THE ONE YOU WANT**
**What it contains:**
- Everything from predictions
- **PLUS** sportsbook odds/lines
- **PLUS** edge calculations (model_prob - implied_prob)
- **PLUS** EV calculations
- Filtered to props where you actually have betting lines

**When it's created:**
- Generated by `props-edges` command
- Combines props_predictions with odds from OddsAPI or Bovada
- Only includes props where you have both model prediction AND a betting line

**Example data (from Oct 17):**
```
player_name         stat  line  model_prob  edge      bookmaker
LaMelo Ball         ast   5.5   0.799767    0.323576  fanduel
Stephon Castle      ast   5.5   0.599551    0.309696  draftkings  
Shai Gilgeous-Alex  ast   5.5   0.718927    0.284144  fanduel
```

**Stats**: 173 props for 38 players (pts, reb, ast)

### 3. `props_recommendations_YYYY-MM-DD.csv` (Filtered Edges)
**What it contains:**
- Subset of props_edges
- Only props that meet recommendation criteria (e.g., edge > threshold, EV > minimum)
- Used for actual betting decisions

**Example filters:**
- Edge > 5%
- EV > $0.50 (or whatever your threshold is)
- Minimum model confidence

## Why Oct 21 Has No Props Edges

The Oct 21 daily update showed:
```
Props edges rows: 0
No edges computed (missing odds or predictions)
```

**Root cause:** The `predict-props` command with `--slate-only` tries to get the game slate from the NBA scoreboard, but the scoreboard doesn't return data for Oct 21 yet (too far in advance).

**Result:** 
- ✅ `props_predictions_2025-10-21.csv` created but empty (0 players)
- ❌ `props_edges_2025-10-21.csv` not created (no predictions to combine with odds)
- ❌ `props_recommendations_2025-10-21.csv` empty (no edges to recommend)

## The Solution

### Option 1: Wait Until Game Day
The scoreboard will have slate data closer to tipoff, so the daily update will work automatically.

### Option 2: Use Historical Data (Requires PyArrow)
Run `predict-props` without `--slate-only` to predict for all players from historical data:
```powershell
predict-props --date 2025-10-21 --no-slate-only --use-pure-onnx
```

**Issue**: This requires pyarrow to read the features parquet file, which can't be installed on ARM64 Windows.

### Option 3: Manual Slate (Advanced)
Create a manual slate CSV with the teams/players playing on Oct 21, then modify the code to read from that file.

## Pipeline Flow for Props

```
1. predict-props (NPU ONNX models)
   ↓
   Generates: props_predictions_YYYY-MM-DD.csv
   (Raw model values for all players)
   
2. Fetch odds (OddsAPI or Bovada)
   ↓
   Gets: Betting lines and prices
   
3. props-edges (Combine predictions + odds)
   ↓
   Generates: props_edges_YYYY-MM-DD.csv ⭐
   (Model predictions + odds + edge calculations)
   
4. export-props-recommendations (Filter by criteria)
   ↓
   Generates: props_recommendations_YYYY-MM-DD.csv
   (Best bets only)
```
